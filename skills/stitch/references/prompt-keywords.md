# Stitch Prompt Keywords

Use these keywords to make Stitch outputs more consistent and controllable.

## Layout Keywords

- `two-panel layout`
- `left navigation rail`
- `sticky header`
- `card-based content grid`
- `table with inline filters`
- `split view with inspector`
- `stepper onboarding flow`
- `empty state with primary CTA`

## Visual Direction Keywords

- `minimal and clean`
- `dark modern interface`
- `high contrast enterprise`
- `editorial landing style`
- `playful consumer product`
- `premium restrained palette`
- `soft rounded components`
- `sharp geometric components`

## Content Hierarchy Keywords

- `clear visual hierarchy`
- `action-first controls`
- `scannable summary cards`
- `progressive disclosure`
- `focus on critical metrics`
- `de-emphasize secondary metadata`

## Interaction Keywords

- `inline validation states`
- `keyboard-friendly navigation`
- `sticky primary actions`
- `contextual quick actions`
- `guided empty states`
- `error recovery affordances`

## Responsive Keywords

- `mobile-first stacking`
- `collapse sidebar to drawer`
- `preserve key CTA above fold`
- `adaptive card-to-list transition`

## Accessibility Keywords

- `wcag aa contrast`
- `visible focus ring`
- `semantic status indicators`
- `touch-friendly targets`

## Prompt Assembly Recipe

1. Start with page goal and user role.
2. Add one layout keyword + one visual direction keyword.
3. Add 2-3 hierarchy/interaction keywords.
4. Add responsive + accessibility constraints.
5. End with explicit output expectation.

## Example Compose

```text
Design a B2B analytics dashboard for operations managers.
Use a two-panel layout with left navigation rail and sticky header.
Style: high contrast enterprise, clear visual hierarchy, scannable summary cards.
Interactions: inline validation states, contextual quick actions.
Responsive: collapse sidebar to drawer on mobile.
Accessibility: wcag aa contrast and visible focus ring.
Return production-ready UI screen with realistic data labels.
```
